# siasgfacil-app Scaffold Design

## Goal

Create a new AI-first project named `siasgfacil-app` under `/home/coringawc/projects` using the local `ai-first-kit` checkout and its `laravel-filament` stack pack.

## Decisions

- Use `/home/coringawc/projects` as the parent directory because `/coringawc/projects` does not exist on this host.
- Use the `laravel-filament` pack from `/home/coringawc/projects/ai-first-kit/packs/laravel-filament`.
- Create a local git repository on branch `main` with an initial commit.
- Do not create or push to a GitHub remote in this step.

## Architecture

The scaffold is generated by `scripts/init-project.sh` instead of manually copying files. The script owns validation, destination checks, base AI-first layout creation, pack skill copying, and local git initialization.

The resulting project path is:

```text
/home/coringawc/projects/siasgfacil-app
```

Expected baseline contents:

- `README.md` with the selected pack description.
- `.agents/skills/` populated from the `laravel-filament` pack.
- `.cert/` and `.git-crypt-keys/` guard directories.
- `.gitattributes` with git-crypt template lines.
- `.gitignore` with common dependency, environment, OS, and log ignores.
- Git repository initialized on `main` with one scaffold commit.

## Data Flow

Execution passes configuration through environment variables:

- `AFK_KIT_DIR=/home/coringawc/projects/ai-first-kit`
- `AFK_INIT_PACK=laravel-filament`
- `AFK_INIT_NAME=siasgfacil-app`
- `AFK_INIT_PARENT_DIR=/home/coringawc/projects`
- `AFK_INIT_COMMIT=1`
- `AFK_INIT_PUSH=0`

The script creates the parent directory if needed, creates the project directory, copies pack-owned skills, initializes git, commits the scaffold, and prints a JSON summary.

## Error Handling

The script should abort before modifying an existing non-empty destination. If `/home/coringawc/projects/siasgfacil-app` already exists and is not empty, stop and ask for direction instead of deleting or overwriting it.

If git commit fails because user identity is missing, leave the generated files intact and report the exact blocker.

## Verification

After running the scaffold command, verify:

- The destination directory exists.
- Git status is clean inside the new project.
- The latest commit exists and matches the initial scaffold intent.
- `README.md` mentions `siasgfacil-app` and pack `laravel-filament`.
- `.agents/skills/init-project/SKILL.md` exists, confirming pack skills were copied.

## Out of Scope

- Installing Laravel dependencies.
- Running the pack-local `/init-project` skill.
- Creating a GitHub repository.
- Enabling git-crypt or configuring certificates.
