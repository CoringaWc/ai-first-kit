# Docker Stack Octane (Swoole)

## Consistency First

A stack canônica deste perfil roda Octane com Swoole na porta interna `8000`, atrás de Nginx que faz `proxy_pass`. Use apenas em projetos **novos**; para migrar um projeto FPM existente, vá em `enable-octane-swoole`.

Base image canônica: `phpswoole/swoole:php8.4` (vem com `ext-swoole` compilado). **Não** rodar `pecl install swoole` em cima — duplica a extensão e o worker recusa boot. Libs de SO, Node 22 e extensões PHP comuns ficam em `_shared/docker-base-deps.md`; sempre que mudar versão de Node ou Chromium libs, edite o shared e propague.

Se já existe `docker-compose.yml` customizado (sem o header desta skill e sem marcador do Sail), pare e peça confirmação antes de sobrescrever.

## Quick Reference

- Para qualquer alteração em `/vite`, `/ws`, `/app`, HMR, Reverb ou Nginx, aplicar antes `vite-reverb-nginx-routing`.
- Base: `phpswoole/swoole:php8.4`.
- Comando do `app`: `php artisan octane:start --server=swoole --host=0.0.0.0 --port=8000`.
- Hot reload em dev: variável `OCTANE_WATCH=true` lida pelo entrypoint condicional; adiciona `--watch` ao comando.
- Services obrigatórios: `app`, `queue`, `reverb`, `vite`, `nginx`, `postgres`, `redis`, `mailpit`.
- Services opcionais (profiles desligados por padrão): `minio`, `ollama`, `qdrant`.
- Mesma imagem em `app`/`queue`/`reverb`/`vite` com comandos diferentes; sem Supervisor.
- Nginx expõe Vite apenas em `/vite/` e Reverb apenas em `/ws/`; não publique a porta `5173` para o browser, bloqueie `/ws/apps/` publicamente e não reserve `/app` para WebSocket.
- Portas de banco, Redis, Mailpit e serviços opcionais devem bindar em `127.0.0.1` por padrão.
- Env vars Octane: `OCTANE_SERVER=swoole`, `OCTANE_HTTPS=true`, `OCTANE_MAX_REQUESTS=500`, `OCTANE_WATCH=false` (dev pode ligar).
- `.env` canônico: `DB_HOST=postgres`, `APP_SERVICE=app`, `WWWUSER=1000`, `WWWGROUP=1000`.
- Build antes do Laravel/Sail existir: `docker compose build app`.
- Build depois do Sail existir: `vendor/bin/sail build --no-cache`.

## Workflow

- [ ] Detectar variáveis (recebidas por `init-project` ou perguntar): `LOCALE_FULL` (`pt_BR.UTF-8` por padrão), `PROJECT_NAME` (basename do diretório).

- [ ] Arquivar compose padrão do Sail antes de gerar a stack do kit:

```bash
COMPOSE_FILES=(compose.yaml compose.yml docker-compose.yml docker-compose.yaml)
EXISTING_COMPOSE_FILES=()
CANONICAL_COMPOSE=false
CUSTOM_COMPOSE_FILES=()

for COMPOSE_FILE in "${COMPOSE_FILES[@]}"; do
  [[ -f "$COMPOSE_FILE" ]] && EXISTING_COMPOSE_FILES+=("$COMPOSE_FILE")
done

for COMPOSE_FILE in "${EXISTING_COMPOSE_FILES[@]}"; do
  if [[ "$COMPOSE_FILE" == "docker-compose.yml" ]] && grep -q "generated by docker-stack-octane-swoole" "$COMPOSE_FILE"; then
    continue
  fi
  grep -Eq "laravel\.test|vendor/laravel/sail" "$COMPOSE_FILE" || CUSTOM_COMPOSE_FILES+=("$COMPOSE_FILE")
done

if ((${#EXISTING_COMPOSE_FILES[@]} > 0)); then
  if ((${#CUSTOM_COMPOSE_FILES[@]} > 0)); then
    printf 'Compose customizado detectado (%s). Migre via enable-octane-swoole.\n' "${CUSTOM_COMPOSE_FILES[*]}"
    exit 1
  elif [[ -f docker-compose.yml ]] && grep -q "generated by docker-stack-octane-swoole" docker-compose.yml; then
    echo "docker-compose.yml canônico Octane detectado; reexecutando idempotente."
    CANONICAL_COMPOSE=true
  elif grep -Eq "laravel\.test|vendor/laravel/sail" "${EXISTING_COMPOSE_FILES[@]}"; then
    if [[ "${ALLOW_SAIL_COMPOSE_REPLACE:-0}" == "1" ]]; then
      echo "Compose padrão do Sail detectado; substituindo pela stack Octane do kit."
    else
      printf 'Compose com marcadores Sail detectado (%s). Sobrescrever? [y/N] ' "${EXISTING_COMPOSE_FILES[*]}"
      read -r ANSWER
      [[ "$ANSWER" =~ ^[Yy]$ ]] || { echo "Abortando geração da stack Octane."; exit 1; }
    fi
  else
    printf 'Compose existente detectado (%s). Migre via enable-octane-swoole.\n' "${EXISTING_COMPOSE_FILES[*]}"
    exit 1
  fi

  TS="$(date +%Y%m%d%H%M%S)"
  for COMPOSE_FILE in "${EXISTING_COMPOSE_FILES[@]}"; do
    [[ "$CANONICAL_COMPOSE" == "true" && "$COMPOSE_FILE" == "docker-compose.yml" ]] && continue
    mv "$COMPOSE_FILE" "${COMPOSE_FILE}.bak.${TS}"
  done
fi
```

mkdir -p docker/php docker/nginx

O `entrypoint.sh` inspeciona `OCTANE_WATCH`: quando `true`, adiciona `--watch`; caso contrário, sobe sem watcher (produção).

- [ ] Normalizar `.env` e `.env.example` (idempotente):

```bash
for KEY in "APP_SERVICE=app" "WWWUSER=1000" "WWWGROUP=1000" \
           "FORWARD_DB_PORT=5432" "FORWARD_REDIS_PORT=6379" \
           "OCTANE_SERVER=swoole" "OCTANE_HTTPS=true" \
           "OCTANE_MAX_REQUESTS=500" "OCTANE_WATCH=false"; do
  KEY_NAME="${KEY%%=*}"
  for ENV_FILE in .env .env.example; do
    [[ -f "$ENV_FILE" ]] || continue
    grep -q "^${KEY_NAME}=" "$ENV_FILE" || echo "$KEY" >> "$ENV_FILE"
  done
done

set_env_entry() {
  local ENV_FILE="$1" ENTRY="$2" MODE="${3:-force}" KEY_NAME="${ENTRY%%=*}"
  [[ "$MODE" == keep && $(grep -c "^${KEY_NAME}=" "$ENV_FILE") -gt 0 ]] && return 0
  if grep -q "^${KEY_NAME}=" "$ENV_FILE"; then
    sed -i "s|^${KEY_NAME}=.*|${ENTRY}|" "$ENV_FILE"
  elif grep -q "^# ${KEY_NAME}=" "$ENV_FILE"; then
    sed -i "s|^# ${KEY_NAME}=.*|${ENTRY}|" "$ENV_FILE"
  else
    echo "$ENTRY" >> "$ENV_FILE"
  fi
}

for ENV_FILE in .env .env.example; do
  [[ -f "$ENV_FILE" ]] || continue
  for ENTRY in "DB_CONNECTION=pgsql" "DB_HOST=postgres" "DB_PORT=5432"; do
    set_env_entry "$ENV_FILE" "$ENTRY"
  done
  for ENTRY in "DB_DATABASE=laravel" "DB_USERNAME=sail" "DB_PASSWORD=password"; do
    set_env_entry "$ENV_FILE" "$ENTRY" keep
  done
done
```

## Anti-Patterns

| Racionalização | Realidade |
| --- | --- |
| "Uso `php:8.4-fpm-bookworm` e instalo Swoole via pecl." | Base oficial `phpswoole/swoole:php8.4` já traz a extensão compilada e testada; pecl em cima duplica. |
| "Rodo `queue` e `reverb` em modo Octane." | Eles usam a mesma imagem mas comandos diferentes (`queue:work`, `reverb:start`). Octane runtime não se aplica a workers de fila e WebSocket. |
| "Coloco Supervisor para tudo dentro do `app`." | Separar lifecycle de HTTP, fila e WebSocket evita restart em cascata e mistura de logs. |
| "Mantenho `DB_HOST=pgsql`." | Service canônico chama `postgres`; manter `pgsql` quebra migrations dentro do container. |
| "Crio bucket MinIO no boot." | O kit não automatiza; faça via seed/bootstrap explícito quando precisar. |
| "Deixo `OCTANE_HTTPS=false` com SSL local ligado." | `route()` gera URLs `http://`; Filament login e redirects quebram. |
| "Não preciso revisar singletons/static; é projeto novo." | Workers ficam vivos entre requests. Mesmo em greenfield, qualquer dependência terceira pode vazar estado entre usuários. |

Sinais ruins: base errada, comando do `app` sem `--host=0.0.0.0`, ausência de entrypoint para `OCTANE_WATCH`, services obrigatórios sem healthcheck, `mailpit` exposto ao público.

## Verification

- [ ] `./.agents/skills/_helpers/verify-skill.sh .agents/skills/docker-stack-octane-swoole/SKILL.md`
- [ ] `docker compose build app` termina sem erro no bootstrap antes do Sail existir.
- [ ] `vendor/bin/sail build` termina sem erro depois do Laravel/Sail existir.
- [ ] `vendor/bin/sail up -d` deixa todos os services obrigatórios em `running`.
- [ ] `docker compose config | grep -q '^  vite:'` retorna 0, e `docker compose config | grep -q '5173:5173'` retorna 1.
- [ ] `grep -q 'location ^~ /vite/' docker/nginx/default.conf && grep -q 'location ^~ /ws/' docker/nginx/default.conf` retorna 0.
- [ ] `vendor/bin/sail exec app php -m | grep -x swoole` retorna `swoole`.
- [ ] `vendor/bin/sail exec app php -m | grep -E '^(mbstring|intl|pdo_pgsql|redis|gd|opcache)$'` lista todas as extensões obrigatórias.
- [ ] `vendor/bin/sail exec app node --version` retorna `v22.*`.
- [ ] `vendor/bin/sail artisan octane:status` mostra workers ativos.
- [ ] `curl -fsSk https://${APP_URL_HOST:-localhost}/` retorna HTTP 200.
