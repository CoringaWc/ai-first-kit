# Recovery de Install Parcial — Passo 0

Cenário: usuário rodou A1 antes, falhou no meio do bootstrap Docker-first, e agora o diretório tem artefatos parciais (`composer.json` sem `vendor/laravel/framework`, `vendor/` incompleto, `docker-compose.yml` parcial, `docker/` parcial, `node_modules/` parcial, ou `boost.json` incompleto).

## Detecção

```bash
if [ -f composer.json ] && [ ! -d vendor/laravel/framework ]; then
  echo "Detectado install parcial de Laravel."
  ls -la
fi

if [ -f docker-compose.yml ] && ! grep -Eq "generated by docker-stack-(fpm|octane-swoole)" docker-compose.yml; then
  echo "Compose existente não-canônico detectado."
  ls -la docker-compose.yml docker/ 2>/dev/null || true
fi
```

## Limpeza (após confirmação do usuário)

**Nunca executar automaticamente.** Apresentar a lista ao usuário e aguardar OK explícito:

```bash
# Após confirmação manual do usuário:
rm -rf vendor/ composer.lock node_modules/ package-lock.json package.json public/ app/ bootstrap/ config/ database/ resources/ routes/ storage/ tests/ .env composer.json artisan boost.json docker/ docker-compose.yml
```

Manter `.git/`, `.agents/`, `.cert/`, `.git-crypt-keys/`, `.gitattributes`, `.gitignore` e `README.md` intocados — eles pertencem ao scaffold ai-first. Se houver commits anteriores, o usuário pode preferir `git clean -fdx` ou checkout para um estado limpo em vez de `rm` seletivo.

## Re-executar

Voltar ao Passo 0 da `SKILL.md`. A verificação aceita o scaffold ai-first e seguirá pelo fluxo Docker-first.
