# Migração FPM → Octane: passos de backup e aplicação de templates

Procedimento mecânico após detecção e branch de backup já criados (ver SKILL.md Workflow passos 1–2).

## 1. Backup `.bak` dos arquivos a sobrescrever

```bash
TS=$(date +%Y%m%d-%H%M%S)
[[ -f docker/php/Dockerfile ]]     && cp docker/php/Dockerfile     "docker/php/Dockerfile.bak.${TS}"
[[ -f docker/nginx/default.conf ]] && cp docker/nginx/default.conf "docker/nginx/default.conf.bak.${TS}"
[[ -f docker-compose.yml ]]        && cp docker-compose.yml        "docker-compose.yml.bak.${TS}"
[[ -f .github/workflows/ci.yml ]]  && cp .github/workflows/ci.yml  ".github/workflows/ci.yml.bak.${TS}"
```

Não pular: `.bak` é o que viabiliza `rollback.md`. Branch git é necessária mas não suficiente — o rollback precisa do snapshot file-level.

## 2. Aplicar templates Octane

```bash
SRC=".agents/skills/docker-stack-octane-swoole/templates"

if ! grep -q "^# generated by docker-stack-fpm" docker-compose.yml 2>/dev/null; then
  diff -u "docker-compose.yml.bak.${TS}" "$SRC/docker-compose.yml" || true
  echo ""
  echo "AVISO: docker-compose.yml customizado. Aplique manualmente o diff da seção 'app':"
  echo "  1. Usar docker/php/Dockerfile atualizado (base phpswoole/swoole:php8.4)"
  echo "  2. Healthcheck do app para porta 8000"
  echo "  3. Nginx faz proxy para app:8000"
  echo "  4. Não publicar porta 8000 externamente"
  exit 1
fi

LOCALE_FULL="${LOCALE_FULL:-${LOCALE:-pt_BR.UTF-8}}"
sed "s/{{LOCALE}}/${LOCALE_FULL}/g" "$SRC/Dockerfile.octane" > docker/php/Dockerfile
cp "$SRC/nginx-default.conf" docker/nginx/default.conf
cp "$SRC/octane-entrypoint.sh" docker/php/entrypoint.sh
chmod +x docker/php/entrypoint.sh

sed -i '1s/generated by docker-stack-fpm/generated by docker-stack-octane-swoole/' docker-compose.yml
sed -i "s|fsockopen('127.0.0.1', 9000)|fsockopen('127.0.0.1', 8000)|" docker-compose.yml
```

## 3. Detecção de customização

O `if ! grep -q "^# generated by docker-stack-fpm"` aborta quando o compose perdeu o header de geração — sinal de edição manual. Nesse caso, **não** sobrescrever: gerar diff, abortar, exigir merge manual da seção `app`. O rollback continua disponível via branch git.

## 4. Pitfalls

- ❌ Sobrescrever `docker-compose.yml` customizado sem diff manual. Quebra produção sem aviso.
- ❌ Aplicar templates sem `.bak` prévio. Branch git ajuda, mas `.bak` permite restaurar arquivo isolado sem `git checkout`.
- ❌ Esquecer `chmod +x docker/php/entrypoint.sh`. Container falha no boot.
- ❌ Aplicar `sed` no `fsockopen` antes de copiar o `Dockerfile.octane` novo. Ordem matters: troca de porta vem por último.
